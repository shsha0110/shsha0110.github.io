1. Introduction: 비선형성(Non-linearity)의 필요성

  • 기계학습 모델을 설계할 때 우리가 마주하는 가장 일반적인 문제 중 하나는 데이터가 선형적인 관계를 가지지 않는다는 것입니다. 주어진 학습 샘플 세트 \(\{(x_1, y_1), \dots, (x_k, y_k)\}\)가 있을 때, \(x\)와 \(y\)의 관계가 단순한 직선이 아니라 곡선 형태를 띨 수 있습니다.

  • 예를 들어, 데이터에 3차 함수 \(y = \theta_3x^3 + \theta_2x^2 + \theta_1x + \theta_0\)를 적합(fitting)시켜야 하는 비선형 회귀(Non-linear Regression) 문제를 생각해 봅시다. 1차원 입력 변수 \(x\) 하나만으로는 선형 모델을 통해 이 곡선을 표현할 수 없습니다. 이러한 한계를 극복하기 위해 등장한 개념이 바로 입력 공간을 확장하는 것입니다.

2. Core Concepts: 특징 사상(Feature Maps)

  • 비선형 문제를 해결하기 위한 직관적인 방법은 원래의 입력 데이터를 더 높은 차원의 공간으로 변환하는 것입니다. 우리는 이를 “다른 특징 변수들의 집합에 대한 선형 함수”로 간주하여 문제를 풀 수 있습니다.

  • 이를 수학적으로 정의하기 위해 매핑 함수 \(\phi\)를 도입합니다. 1차원 실수 공간 \(\mathbb{R}\)에서 4차원 실수 공간 \(\mathbb{R}^4\)로 변환하는 함수를 다음과 같이 정의해 봅시다:

\[ \phi(x) = \begin{bmatrix} 1 \\ x \\ x^2 \\ x^3 \end{bmatrix} \in \mathbb{R}^4 \]

  • 이러한 변환을 거치면, 원래의 3차 함수 방정식은 다음과 같이 매개변수 벡터 \(\theta\)와 매핑된 벡터 \(\phi(x)\)의 내적(Inner product)으로 간단히 다시 쓸 수 있습니다:

\[ \theta_3x^3 + \theta_2x^2 + \theta_1x + \theta_0 = \theta^\top \phi(x) \]

  • 이때 사용하는 용어를 명확히 정리할 필요가 있습니다. 원래의 입력 값인 \(x\)를 입력 속성(Input attributes)이라고 부릅니다. 그리고 함수 \(\phi\)를 통해 새롭게 매핑된 벡터 \(\phi(x)\)를 특징 변수(Feature variables)라고 부르며, 이 변환 함수 \(\phi\) 자체를 특징 사상(Feature map)이라고 명명합니다.

3. Mathematical Formulation: 고차원 매핑의 한계

3.1. 최소 평균 제곱(LMS)과 특징 사상의 결합

  • 특징 사상을 실제 학습 알고리즘에 어떻게 적용할 수 있을까요? 기본 형태인 최소 제곱치 배치 경사 하강법(Least-Square Batch Gradient Descent)의 업데이트 규칙을 살펴보며 그 수학적 배경을 짚어보겠습니다.

  • 원래 공간에서의 파라미터 업데이트 식은 다음과 같습니다: \[ \theta = \theta + \alpha \sum_{i=1}^{n} (y^{(i)} - \theta^\top x^{(i)})x^{(i)} \]

    • \(n\)은 학습 데이터의 개수를 의미합니다.
    • \(x^{(i)}\)는 \(i\)번째 데이터를 의미합니다.

[원래 공간(\(x\))에서의 유도 과정]

    1. 비용 함수(Cost Function) 정의:
    • 선형 회귀에서 예측 함수는 \(h_\theta(x^{(i)}) = \theta^\top x^{(i)}\)입니다. 전체 \(n\)개의 데이터에 대한 오차 제곱합을 비용 함수 \(J(\theta)\)로 정의합니다. 미분의 편의를 위해 앞에 상수를 곱해줍니다. \[ J(\theta) = \frac{1}{2} \sum_{i=1}^{n} \left( \theta^\top x^{(i)} - y^{(i)} \right)^2 \]
    1. 그래디언트(Gradient) 계산:
    • 비용 함수를 최소화하기 위해 \(J(\theta)\)를 \(\theta\)에 대해 편미분합니다. 연쇄 법칙(Chain Rule)에 의해 다음과 같이 정리됩니다. \[ \nabla_\theta J(\theta) = \sum_{i=1}^{n} \left( \theta^\top x^{(i)} - y^{(i)} \right) x^{(i)} \]
    1. 경사 하강법(Gradient Descent) 업데이트:
    • 기울기의 반대 방향으로 학습률(Learning rate) \(\alpha\)만큼 이동하여 파라미터를 업데이트합니다. 식을 전개하고 부호를 정리하면 위에서 제시한 기본 업데이트 식이 완성됩니다. \[ \theta = \theta - \alpha \nabla_\theta J(\theta) = \theta + \alpha \sum_{i=1}^{n} \left( y^{(i)} - \theta^\top x^{(i)} \right) x^{(i)} \]

[특징 공간(\(\phi(x)\))으로의 확장]

  • 이제 원래의 데이터 \(x\) 대신, 특징 사상 \(\phi: \mathbb{R}^d \rightarrow \mathbb{R}^p\)를 도입하여 파라미터 벡터를 \(\theta \in \mathbb{R}^p\)로 확장해 봅시다.

  • 구조적으로 예측 함수에 들어가는 입력값만 \(x^{(i)}\)에서 \(\phi(x^{(i)})\)로 바뀌었을 뿐, 선형 결합이라는 본질은 완전히 동일합니다. 따라서 비용 함수를 미분할 때 연쇄 법칙의 속미분 결과로 \(x^{(i)}\) 대신 \(\phi(x^{(i)})\)가 도출됩니다.

  • 결과적으로 기존 알고리즘의 구조를 전혀 바꾸지 않고, 데이터가 들어가는 자리에 \(x\) 대신 \(\phi(x)\)를 치환함으로써 업데이트 식은 자연스럽게 다음과 같이 변형됩니다:

\[ \theta = \theta + \alpha \sum_{i=1}^{n} (y^{(i)} - \theta^\top \phi(x^{(i)}))\phi(x^{(i)}) \]

3.2. 차원의 저주와 계산 비용 문제

  • 위 식은 논리적으로 완벽하지만, 치명적인 계산적 결함을 내포하고 있습니다. 만약 변환된 특징 공간의 차원(\(p\))이 매우 커진다면, \(\phi(x)\)를 명시적으로 계산하는 데 엄청난 연산 비용이 발생합니다.

  • 극단적인 예를 들어보겠습니다. 입력 벡터 \(x \in \mathbb{R}^d\)가 주어졌을 때, 차수가 3 이하인 모든 단항식(monomials)을 포함하는 벡터 \(\phi(x)\)를 구성한다고 가정해 봅시다. 이 벡터 안에는 \(1, x_1, x_2, \dots, x_1^2, x_1 x_2, \dots, x_1^3, x_1 x_2 x_3\) 등 무수한 조합이 포함됩니다.

  • 이 매핑된 벡터의 차원은 대략 \(d^3\)에 비례하게 됩니다(\(O(d^3)\)). 만약 원래 데이터의 차원 \(d = 10^3\)이라면, 변환된 벡터 \(\phi(x)\)는 무려 \(10^9\) (10억) 차원을 가지게 됩니다. 이는 메모리에 저장하는 것조차 버거울 뿐만 아니라, 일반적인 최소 제곱법 알고리즘보다 \(10^6\)배 (100만 배) 이상 느려지게 만듭니다.

4. Detailed Derivations: 커널 트릭(Kernel Trick)의 유도

  • 이처럼 매핑된 벡터의 차원 \(p \approx d^3\) 수준으로 비대해지는 문제를 피할 수 없을까요? 결론부터 말하자면, 커널 트릭(Kernel Trick)을 사용하면 고차원 벡터 \(\phi(x)\)를 직접 계산하지 않고도 동일한 결과를 얻을 수 있습니다.

4.1. 핵심 보조정리 (Lemma)와 증명

  • 커널 트릭의 핵심은 다음과 같은 보조정리(Lemma)에서 출발합니다:

“파라미터 벡터 \(\theta\)는 항상 학습 데이터들의 특징 벡터 \(\phi(x^{(1)}), \dots, \phi(x^{(n)})\)의 선형 결합(Linear combination)으로 표현될 수 있다.”

  • 이 명제는 수학적 귀납법(Mathematical induction)을 통해 완벽하게 증명할 수 있습니다.

[Step 1: 기저 조건 (Initialization)]

  • 간단한 증명을 위해 파라미터의 초기값을 \(\theta = 0\)으로 가정합니다. 이는 명백하게 \(\phi(x^{(i)})\)들의 선형 결합입니다. \[ \theta = 0 = \sum_{i=1}^{n} 0 \cdot \phi(x^{(i)}) \]

[Step 2: 귀납적 가정 (Inductive Hypothesis)]

  • 학습의 특정 시점에서 \(\theta\)가 어떤 실수 계수 \(\beta_1, \dots, \beta_n \in \mathbb{R}\)에 대해 다음과 같이 표현된다고 가정합시다: \[ \theta = \sum_{i=1}^{n} \beta_i \phi(x^{(i)}) \]

[Step 3: 귀납적 단계 (Inductive Step)]

  • 경사 하강법의 다음 라운드에서 업데이트된 \(\theta\) 식에 가정을 대입합니다: \[ \theta_{new} = \theta + \alpha \sum_{i=1}^{n} (y^{(i)} - \theta^\top \phi(x^{(i)}))\phi(x^{(i)}) \]

\[ = \sum_{i=1}^{n} \beta_i \phi(x^{(i)}) + \alpha \sum_{i=1}^{n} (y^{(i)} - \theta^\top \phi(x^{(i)}))\phi(x^{(i)}) \quad \text{} \]

  • 이 식을 \(\phi(x^{(i)})\)로 묶어 정리하면 다음과 같습니다:

\[ = \sum_{i=1}^{n} \left( \beta_i + \alpha (y^{(i)} - \theta^\top \phi(x^{(i)})) \right) \phi(x^{(i)}) \]

  • 괄호 안의 값이 새로운 계수 \(\beta_i^{new}\)가 됩니다. 즉, 업데이트 이후의 \(\theta\) 역시 여전히 \(\phi(x^{(1)}), \dots, \phi(x^{(n)})\)의 선형 결합 형태를 유지함을 알 수 있습니다.

\[ = \sum_{i=1}^{n} \beta_i^{new} \phi(x^{(i)}) \]

4.2. 커널(Kernel)의 정의와 업데이트 규칙의 재구성

  • 보조정리가 참이므로, 우리는 이제 고차원 벡터 \(\theta\)를 직접 업데이트하는 대신, \(n\)개의 스칼라 계수 \(\beta_1, \dots, \beta_n\)만을 업데이트하는 문제로 변환할 수 있습니다.

  • 새로운 업데이트 식은 모든 \(i \in \{1, \dots, n\}\)에 대해 다음과 같습니다: \[ \beta_i = \beta_i + \alpha \left( y^{(i)} - \sum_{j=1}^{n} \beta_j \phi(x^{(j)})^\top \phi(x^{(i)}) \right) \]

  • 여기서 주목해야 할 부분은 \(\phi(x^{(j)})^\top \phi(x^{(i)})\)라는 내적(inner product) 항입니다. 이 내적 연산을 간결하게 표기하기 위해 \(\langle \phi(x^{(j)}), \phi(x^{(i)}) \rangle\) 기호를 사용하거나, 커널 함수(Kernel function) \(K(x^{(i)}, x^{(j)})\)로 정의합니다. 즉, 특징 사상에 대응하는 커널 함수 \(K: \mathcal{X} \times \mathcal{X} \rightarrow \mathbb{R}\)는 다음과 같이 정의됩니다. \[ K(x, z) := \langle \phi(x), \phi(z) \rangle \]

4.3. 커널 트릭은 왜 효율적인가?

  • 매 반복(iteration)마다 모든 내적 \(\langle \phi(x^{(j)}), \phi(x^{(i)}) \rangle\)을 계산하는 것은 여전히 특징 공간의 차원인 \(O(p)\) 만큼의 연산을 필요로 하는 것처럼 보일 수 있습니다. 그러나 이 방법은 두 가지 결정적인 이유로 인해 매우 효율적입니다.

      1. 사전 계산(Pre-computation): 루프를 돌기 전에 모든 \(i, j\) 쌍에 대해 내적 값을 미리 계산하여 행렬 형태로 저장해 둘 수 있습니다.
      1. 연산의 우회(Efficient calculation): 많은 특징 사상의 경우, 명시적으로 \(\phi\) 벡터를 전개하지 않고도 커널 값 자체를 매우 빠르게 계산할 수 있는 수학적 트릭이 존재합니다.

  • 예를 들어 원래 차원이 \(d\)이고, 3차 이하의 모든 단항식을 포함하는 매핑의 내적은 다음과 같이 정리될 수 있습니다: \[ \langle \phi(x), \phi(z) \rangle = 1 + \sum_{i=1}^{d} x_i z_i + \sum_{i,j} x_i x_j z_i z_j + \sum_{i,j,k} x_i x_j x_k z_i z_j z_k \] \[ = 1 + \sum_{i=1}^{d} x_i z_i + \left(\sum_{i=1}^{d} x_i z_i\right)^2 + \left(\sum_{i=1}^{d} x_i z_i\right)^3 \] \[ = 1 + \langle x, z \rangle + \langle x, z \rangle^2 + \langle x, z \rangle^3 \]

  • 이 결과를 보면, 고차원 벡터의 내적이 원래 \(d\)차원 공간에서의 단순한 내적 \(\langle x, z \rangle\) 연산만으로 귀결됩니다! 따라서 \(O(d^3)\)이 걸리던 연산이 단 \(O(d)\) 시간 만에 완료됩니다.

5. Algorithm & Kernel Examples

  • 위의 논리를 종합한 최종 커널 알고리즘은 다음과 같습니다. 우리는 특징 사상 \(\phi(\cdot)\)를 커널 함수 \(K(\cdot, \cdot)\) 안에 캡슐화(encapsulate)하여 \(\phi(x)\)를 명시적으로 계산할 필요를 완전히 없앴습니다.

    1. 모든 \(i, j \in \{1, \dots, n\}\)에 대해 커널 행렬 요소를 계산합니다: \(K(x^{(i)}, x^{(j)}) := \langle \phi(x^{(i)}), \phi(x^{(j)}) \rangle\)
    1. 다음 식을 반복(Loop)하여 계수를 업데이트합니다: \(\beta_i = \beta_i + \alpha (y^{(i)} - \sum_{j=1}^{n} \beta_j K(x^{(i)}, x^{(j)}))\)

  • 벡터 표기법(Vector notation)을 사용하면 이 업데이트 규칙은 더욱 직관적으로 표현됩니다: \[ \beta = \beta + \alpha(\vec{y} - K\beta) \quad \text{} \]

5.1. 다항식 커널 (Polynomial Kernels)

  • 위에서 본 효율성의 구체적인 예시들을 확인해보겠습니다. \(x, z \in \mathbb{R}^d\)라고 할 때, 다음과 같은 커널들을 생각해볼 수 있습니다.

(1) 제곱 커널: \(K(x, z) = (x^\top z)^2\)

  • 이를 풀어서 쓰면 다음과 같습니다: \[ K(x, z) = \left( \sum_{i=1}^{d} x_i z_i \right) \left( \sum_{j=1}^{d} x_j z_j \right) = \sum_{i=1}^{d} \sum_{j=1}^{d} x_i x_j z_i z_j = \sum_{i,j=1}^{d} (x_i x_j)(z_i z_j) \quad \text{} \]

  • 이 식은 \(\phi(x)\)가 \(x_1x_1, x_1x_2, \dots, x_3x_3\) 등의 항을 가지는 특징 벡터의 내적과 완벽히 일치함을 보여줍니다 . 명시적 사상은 \(O(d^2)\) 시간이 걸리지만, 커널 연산 \((x^\top z)^2\)은 \(O(d)\) 선형 시간만 소요됩니다.

(2) 절편이 포함된 제곱 커널: \(K(x, z) = (x^\top z + c)^2\)

  • 이를 전개하면 2차항, 1차항, 상수항을 모두 포괄하는 특징 공간을 암시합니다: \[ K(x, z) = \sum_{i,j=1}^{d} (x_i x_j)(z_i z_j) + \sum_{i=1}^{d} (\sqrt{2c} x_i)(\sqrt{2c} z_i) + c^2 \]

  • 여기에 대응하는 \(\phi(x)\) 벡터는 기존 2차항들에 \(\sqrt{2c}x_1, \sqrt{2c}x_2, \sqrt{2c}x_3\) 와 상수 \(c\)가 추가된 형태가 됩니다 .

6. Properties of Kernels (Mercer”s Theorem)

  • 그렇다면 우리가 임의로 만든 함수 \(K(x, z)\)가 유효한 커널(어떤 특징 사상 \(\phi\)에 대응되는 커널)인지 어떻게 확인할 수 있을까요? 특징 사상을 해석적으로 접근하지 않고 커널 함수만 단독으로 사용하기 위해서는 특정 수학적 조건을 만족해야 합니다.

  • 이론적으로 커널 행렬(Kernel Matrix)이 대칭(Symmetric)이고 양의 준정부호(Positive semidefinite)라는 것은 유효한 커널(Mercer Kernel)이 되기 위한 필요충분조건(Necessary and sufficient condition)입니다.

  • 여기서 필요조건(Necessary condition) 측면을 수학적으로 증명해 보겠습니다. \(K\)가 어떤 \(\phi\)에 대응하는 유효한 커널이라고 가정하고 , 유한한 \(n\)개의 데이터 포인트로 구성된 정방 행렬 \(K \in \mathbb{R}^{n \times n}\) (이때 \((i, j)\) 성분은 \(K_{ij} = K(x^{(i)}, x^{(j)})\))을 정의합시다.

1. 대칭성 (Symmetry) 증명

  • 내적의 교환 법칙에 의해 커널 행렬은 대칭성을 가집니다.

\[ \begin{aligned} K_{ij} = K(x^{(i)}, x^{(j)}) &= \phi(x^{(i)})^\top \phi(x^{(j)}) \\ &= \phi(x^{(j)})^\top \phi(x^{(i)}) \\ &= K(x^{(j)}, x^{(i)}) = K_{ji} \quad \text{} \end{aligned} \]

2. 양의 준정부호 (Positive Semidefinite) 증명

  • 임의의 열 벡터 \(z\)에 대하여 이차 형식(Quadratic form) \(z^\top K z \ge 0\)이 성립해야 합니다. \[ z^\top K z = \sum_{i} \sum_{j} z_i K_{ij} z_j = \sum_{i} \sum_{j} z_i \phi(x^{(i)})^\top \phi(x^{(j)}) z_j \quad \text{} \]

  • 이를 전개하여 원소별로 다시 묶어보겠습니다. \(\phi_k\)를 벡터 \(\phi\)의 \(k\)번째 성분이라고 하면:

\[ \begin{aligned} &= \sum_{i} \sum_{j} z_i \sum_{k} \phi_k(x^{(i)}) \phi_k(x^{(j)}) z_j \\ &= \sum_{k} \sum_{i} \sum_{j} z_i \phi_k(x^{(i)}) \phi_k(x^{(j)}) z_j \\ &= \sum_{k} \left( \sum_{i} z_i \phi_k(x^{(i)}) \right)^2 \ge 0 \quad \text{} \end{aligned} \]

  • 어떤 실수의 제곱의 합은 항상 0보다 크거나 같으므로, 증명이 완료됩니다.

7. Interpretation and Intuition: 유사도 지표로서의 커널

  • 커널 함수 \(K(x, z) = \phi(x)^\top \phi(z)\)의 기하학적 의미는 본질적으로 “내적”입니다.

    • 특징 공간에서 두 벡터 \(\phi(x)\)와 \(\phi(z)\)가 가까울수록, 내적(커널) 값은 커집니다.
    • 반대로 \(\phi(x)\)와 \(\phi(z)\)가 멀리 떨어져 있거나(거의 직교할 경우), 내적 값은 작아집니다.

  • 따라서 커널 함수 \(K(x, z)\)는 두 데이터 \(x\)와 \(z\)가 얼마나 유사한지를 나타내는 직관적인 측정 지표(Similarity measure)로 해석될 수 있습니다.

  • 이러한 직관을 가장 잘 반영하는 대표적인 커널이 바로 가우시안 커널(Gaussian Kernel), 또는 RBF(Radial Basis Function) 커널입니다. \[ K(x, z) = \exp\left(-\frac{||x - z||^2}{2\sigma^2}\right) \quad \text{} \]

  • 수식을 보면, 두 데이터 \(x, z\)가 완벽히 같아서 거리가 0이 되면 커널 값은 1이 되고, 거리가 무한히 멀어지면 커널 값은 0에 수렴합니다. 이는 유효한 커널 조건을 만족합니다.

7.1. 실제 활용 사례 (Applications)

  • 이러한 커널 방법론은 다양한 머신러닝 문제에서 강력한 성능을 발휘합니다.
    • 손글씨 숫자 인식 (256-pixels Digit Recognition): 0부터 9까지의 필기체 이미지(256 픽셀)가 주어졌을 때, 픽셀 간의 공간적 인접성 정보를 모델에 주지 않고 단지 256차원 벡터로만 취급해도 다항식 커널 \(K(x, z) = (x^\top z)^k\)나 가우시안 커널을 기반으로 한 SVM(Support Vector Machine)은 매우 뛰어난 성능을 보여줍니다.
    • 아미노산/문자열 분류 (Amino Acids/String Classification): 입력 데이터가 길이가 서로 다른 문자열일 경우 고정된 크기의 특징 벡터를 만들기 까다롭습니다. 이때 길이기 \(k\)인 하위 문자열(substring)의 등장 횟수를 카운트하는 특징 벡터 \(\phi(x)\)를 정의해 봅시다. 우리는 이 고차원 벡터를 명시적으로 계산하지 않고도, 문자열 매칭 알고리즘을 사용하여 커널 값 \(K(x, z) = \phi(x)^\top \phi(z)\)만을 효율적으로 직접 계산해 낼 수 있습니다.
