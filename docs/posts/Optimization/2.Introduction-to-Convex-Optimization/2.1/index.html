1. Introduction: 왜 볼록성 보존 연산을 알아야 하는가?

  • 최적화 문제, 특히 머신러닝과 계량경제학에서 마주하는 수많은 문제들에서 기저에 깔린 볼록 구조(Convex structures)를 파악하는 것은 매우 중요합니다. 복잡한 목적 함수나 제약 조건이 주어졌을 때, 처음부터 정의를 이용해 볼록성을 증명하는 것은 까다롭고 비효율적일 수 있습니다.

  • 대신, 우리는 몇 가지 기본적인 규칙들을 이해함으로써 특정 집합이나 함수가 볼록(Convex)인지 판별할 수 있습니다. 나아가 이러한 규칙들을 블록처럼 조립하면, 단순하고 기본적인 볼록 집합 및 함수들로부터 아주 복잡한 형태의 새로운 볼록 집합과 함수를 안전하게 구축할 수 있습니다. 이번 포스트에서는 집합과 함수의 볼록성을 보존하는 핵심 연산자들을 정리합니다.

2. Set Operations Preserving Convexity (볼록성을 보존하는 집합 연산)

  • 어떤 집합들이 볼록 집합일 때, 이들에 특정 연산을 취해 만들어진 새로운 집합 역시 볼록 집합이 됨을 보장하는 규칙들입니다.

2.1 Intersection (교집합)

  • 가장 강력하고 자주 쓰이는 성질 중 하나입니다. 임의의(심지어 무한 개일지라도) 볼록 집합들의 교집합은 항상 볼록 집합입니다.
  • 선형 계획법(LP)에서 여러 개의 선형 부등식(반공간, Halfspaces)이 이루는 교집합인 다면체(Polyhedron)가 볼록 집합이 되는 현상을 설명하는 핵심 원리입니다.

2.2 Scaling (스케일링)

  • 주어진 볼록 집합 \(C\)와 실수 \(\alpha \in \mathbb{R}\)에 대하여, 집합의 크기를 비례적으로 확대/축소하거나 방향을 뒤집는 연산입니다. 이 연산 결과 역시 볼록 집합입니다. \[ \alpha C = \{\alpha x : x \in C\} \]

2.3 Minkowski Sum (민코프스키 합)

  • 두 개 이상의 볼록 집합 \(C_1, \dots, C_k \subseteq \mathbb{R}^d\)가 주어졌을 때, 각 집합에서 원소를 하나씩 뽑아 더한 벡터들의 집합을 민코프스키 합이라고 합니다. 이는 기하학적으로 한 집합을 다른 집합의 테두리를 따라 평행 이동시키며 쓸어담은 영역으로 이해할 수 있으며, 이 역시 볼록성을 보존합니다. \[ C_1 + \dots + C_k = \{x^1 + \dots + x^k : x^i \in C_i \text{ for } i=1,\dots,k\} \]

2.4 Cartesian Product (데카르트 곱 / 곱집합)

  • 볼록 집합 \(C_i \subseteq \mathbb{R}^{d_i} (i=1,\dots,k)\) 들의 데카르트 곱 역시 볼록 집합입니다. 이는 독립적인 여러 제약 조건들이 결합된 상태(예: \(x \in C_1\), \(y \in C_2\))를 하나의 통합된 벡터 공간상에서 다룰 때 유용합니다. \[ C_1 \times \dots \times C_k = \{(x^1, \dots, x^k) \in \mathbb{R}^{d_1} \times \dots \times \mathbb{R}^{d_k} : x^i \in C_i \text{ for } i=1,\dots,k\} \]

2.5 Affine Image & Inverse Affine Image (아핀 상과 역상)

  • 볼록 집합은 선형 변환 및 평행 이동(Affine mapping)을 거쳐도 그 형태적 특성인 볼록성을 잃지 않습니다. 어떤 행렬 \(A \in \mathbb{R}^{p \times d}\)와 벡터 \(b \in \mathbb{R}^p\)로 정의되는 아핀 함수 \(f(x) = Ax + b : \mathbb{R}^d \rightarrow \mathbb{R}^p\)가 있다고 합시다.

  • Affine image (아핀 상): 볼록 집합 \(C\)를 \(f\)를 통해 변환한 결과는 볼록 집합입니다. \[f(C) = \{Ax+b : x \in C\}\]

  • Inverse affine image (아핀 역상): 아핀 변환 결과가 어떤 볼록 집합 \(C\)에 속하게 만드는 원래 \(x\)들의 집합 또한 볼록 집합입니다. \[f^{-1}(C) = \{x : Ax+b \in C\}\]

3. Function Operations Preserving Convexity (볼록성을 보존하는 함수 연산)

  • 목적 함수가 볼록 함수인지 검증하는 데 필수적인 규칙들입니다. 여러 함수를 조합하여 복잡한 머신러닝 손실 함수를 설계할 때 핵심적인 역할을 합니다.

3.1 Nonnegative Weighted Sum (비음수 가중합)

  • 볼록 함수 \(f_1, \dots, f_k : \mathbb{R}^d \rightarrow \mathbb{R}\)에 대하여, 각 함수에 0 이상의 가중치 \(\alpha_1, \dots, \alpha_k \ge 0\)를 곱해 모두 더한 함수는 볼록 함수입니다. \[ \alpha_1 f_1 + \dots + \alpha_k f_k \]

정규화(Regularization) 기법에서 손실 함수 \(L(x)\)와 페널티 항 \(R(x)\)가 모두 볼록일 때, 이들의 합 \(L(x) + \lambda R(x)\) (\(\lambda \ge 0\)) 역시 볼록임을 보장해 줍니다.

3.2 Pointwise Maximum (점별 최댓값)

  • 볼록 함수들의 집합 \(\{f_{\gamma}\}_{\gamma \in \Gamma}\)이 주어졌을 때, 각 지점에서 가장 큰 함수값을 취하여 만든 새로운 함수는 볼록 함수입니다. 이때 함수의 개수를 나타내는 인덱스 집합 \(\Gamma\)는 유한 개일 필요 없이 무한 개(Infinite)여도 무방합니다. \[ f(x) = \max_{\gamma \in \Gamma} f_{\gamma}(x) \]

3.3 Minimizing Out Variables (변수 소거 / 부분 최소화)

  • 함수 \(g(x, y)\)가 변수 쌍 \((x, y)\) 모두에 대해 볼록 함수라고 가정합시다. 어떤 볼록 집합 \(C\)에 속하는 \(y\)에 대해 \(g(x,y)\)를 최소화하여 변수 \(y\)를 소거해버리면, 오직 \(x\)에 대한 함수 \(f(x)\)를 얻게 됩니다. \[ f(x) = \inf_{y \in C} g(x, y) \]
  • 놀랍게도, 이렇게 만들어진 함수 \(f(x)\) 역시 \(x\)에 대한 볼록 함수가 됩니다.

3.4 Perspective Function (투시 함수 / 원근 함수)

  • \(g(x)\)가 볼록 함수일 때, 이를 바탕으로 차원이 하나 추가된 변수 쌍 \((x, t) \in \mathbb{R}^d \times \mathbb{R}_{++}\) 공간에서 다음처럼 정의된 함수 \(f(x,t)\)는 볼록 함수입니다. 이를 \(g\)의 투시 함수라 부릅니다. \[ f(x, t) = t \cdot g(x/t) \]

3.5 Composition (함수의 합성)

  • 함수를 합성할 때 볼록성이 유지되기 위해서는 함수의 증감 조건이 함께 고려되어야 합니다.
    • Affine Composition (아핀 합성): 어떤 볼록 함수 \(g: \mathbb{R}^p \rightarrow \mathbb{R}\)의 입력으로 아핀 함수 \(Ax+b\)를 넣은 경우, \(f(x) = g(Ax+b)\)는 \(x\)에 대해 무조건 볼록입니다.
    • General Composition (일반적인 합성): 일변수 함수 \(h: \mathbb{R} \rightarrow \mathbb{R}\)와 다변수 함수 \(g: \mathbb{R}^d \rightarrow \mathbb{R}\)가 주어졌을 때, \(f = h \circ g\) (즉, \(f(x) = h(g(x))\))가 볼록이 되려면 조건이 필요합니다. \(h\)가 단조 비감소(non-decreasing)하는 볼록 함수이고, \(g\)가 볼록 함수라면 \(f\)는 볼록 함수가 됩니다.

4. Example: Distance Functions (거리 함수 분석)

  • 앞서 배운 규칙들이 실제 문제에 어떻게 적용되는지 확인하기 위해, 특정 위치들의 집합 \(C\)까지의 거리를 측정하는 두 가지 함수를 살펴봅시다.

  • 거리 자체를 나타내는 함수 \(g(x, y) = ||x - y||\)는 \((x, y)\) 모두에 대하여 볼록 함수라는 사실에서 출발합니다.

(1) 가장 먼 거리 측정: \(f_1(x) = \max_{y \in C} ||x-y||\)

  • \(f_1(x)\)는 점 \(x\)로부터 집합 \(C\) 내의 어떤 점까지의 거리 중 가장 긴 거리(longest distance)를 측정합니다.
  • 이 함수는 주어진 여러 점 \(y \in C\)에 대하여 \(||x-y||\)라는 볼록 함수들의 최댓값을 구하는 것과 같습니다. 따라서 앞서 살펴본 ‘Pointwise Maximum’ 성질에 의해, 집합 \(C\)가 볼록 집합인지 아닌지와 무관하게 언제나 볼록 함수가 됩니다.

(2) 가장 짧은 거리 측정: \(f_2(x) = \min_{y \in C} ||x-y||\)

  • 반면 \(f_2(x)\)는 점 \(x\)로부터 집합 \(C\)까지의 가장 짧은 거리(shortest distance)를 구하는 함수입니다.
  • 이는 볼록 함수 \(g(x,y) = ||x-y||\)에 대하여 변수 \(y\)에 대한 부분 최소화(Minimizing out variables)를 수행하는 형태입니다. 이 연산이 \(x\)에 대한 볼록성을 유지하려면 최소화를 수행하는 도메인(여기서는 \(C\))이 반드시 볼록 집합이어야 합니다. 따라서, \(f_2(x)\)는 오직 집합 \(C\)가 볼록 집합일 경우에만 볼록 함수가 됩니다.
