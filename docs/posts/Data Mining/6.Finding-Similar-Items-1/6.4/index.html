1. Minhashing 구현의 난제와 해결책

  • 이전 시간에 배운 Minhashing의 기본 개념은 “전체 행(Row)의 순서를 무작위로 섞은 뒤(Permutation), 처음으로 1이 등장하는 행의 번호를 찾는다”는 것이었습니다. 하지만 이를 현실 세계의 거대한 특성 행렬(Characteristic Matrix)에 그대로 적용하는 것은 불가능합니다.

1.1 명시적 순열(Explicit Permutation)의 한계

  • 만약 웹페이지에서 추출한 고유 토큰이 10억 개 존재하여 행렬의 행이 10억 줄이라고 가정해 봅시다. 이 10억 개의 행을 물리적으로 메모리 상에서 매번 뒤섞고 정렬하는 작업은 계산량(\(O(N \log N)\))과 메모리 접근 비용 측면에서 감당할 수 없는(infeasible) 수준의 오버헤드를 발생시킵니다.

1.2 순열의 근사 (Approximating Permutation)

  • 행을 실제로 뒤섞는 대신, 해시 함수(Hash function)를 활용하여 무작위 순열을 수학적으로 근사(Approximate)하는 매우 영리한 아이디어를 사용합니다.
      1. 원본 행의 인덱스 \(r = 1, \dots, n\)에 대해 무작위 해시 함수 \(h(r)\)을 적용하여 해시값을 계산합니다.
      1. 행을 직접 섞는 대신, 이 해시값 \(h(r)\)을 기준으로 행렬을 정렬한다고 상상해 봅니다.
      1. 이렇게 해시값 순서대로 매겨진 새로운 순서를 완벽하게 무작위로 뒤섞인 순열(Random permutation)과 동치로 간주합니다.
  • 이를 구현하기 위해, 우리는 모든 해시 함수에 대한 최소 해시값을 저장할 슬롯(Slot) 행렬을 만들고, 초기값을 모두 무한대(\(\infty\))로 설정합니다. 이후 행을 위에서부터 차례로 읽어 내려가면서, 데이터가 존재하는(값이 1인) 경우에 한해 기존 슬롯 값보다 현재 해시값이 더 작으면 최소값으로 갱신(Update)하는 방식을 취합니다.

2. Minhashing 최적화 알고리즘

  • 위의 아이디어를 바탕으로 컴퓨터가 효율적으로 처리할 수 있는 형태의 Minhashing 알고리즘은 다음과 같이 작성됩니다.

[Minhashing Algorithm]

  1. \(T\)를 원본 특성 행렬(Characteristic Matrix)이라고 한다.
  2. 서명을 저장할 슬롯 행렬 \(M\)을 초기화한다.
  3. 모든 해시 함수 \(h_i\)와 모든 열 \(c\)에 대하여:
  4. \(\quad M(i, c) \leftarrow \infty\) (무한대로 초기화)
  5. for each 행 \(r\) and 열 \(c\):
  6. \(\quad\) if \(T(r, c) = 1\) then
  7. \(\quad \quad\) for each 해시 함수 \(h_i\):
  8. \(\quad \quad \quad\) if \(h_i(r) < M(i, c)\) then
  9. \(\quad \quad \quad \quad M(i, c) \leftarrow h_i(r)\)
  • 이 알고리즘의 가장 큰 장점은 전체 행렬을 메모리에 올릴 필요 없이, 데이터 스트림처럼 행을 한 줄씩(Row-by-row) 순차적으로 읽으면서 처리할 수 있다는 점입니다.

3. Pop Quiz: 시그니처 행렬 직접 계산해보기

  • 강의 자료에 제시된 퀴즈를 통해 알고리즘이 어떻게 작동하는지 직접 손으로 풀어보며 완벽히 이해해 봅시다.

  • [입력 조건]

    • 해시 함수 1: \(g_1(x) = (x + 1) \bmod 4\)
    • 해시 함수 2: \(g_2(x) = (3x + 1) \bmod 4\)
    • 특성 행렬 \(T\):
Row \(r\) \(S_1\) \(S_2\) \(S_3\) \(g_1(r)\) \(g_2(r)\)
0 1 0 0 1 1
1 0 0 1 2 0
2 0 1 0 3 3
3 1 0 1 0 2
  • [단계별 풀이 과정]
    • 초기 상태: 슬롯 행렬 \(M\)의 모든 원소는 \(\infty\)입니다.
    • Row 0 스캔: \(S_1\)만 1을 가집니다.
      • \(M(g_1, S_1) = \min(\infty, 1) = 1\)
      • \(M(g_2, S_1) = \min(\infty, 1) = 1\)
    • Row 1 스캔: \(S_3\)만 1을 가집니다.
      • \(M(g_1, S_3) = \min(\infty, 2) = 2\)
      • \(M(g_2, S_3) = \min(\infty, 0) = 0\)
    • Row 2 스캔: \(S_2\)만 1을 가집니다.
      • \(M(g_1, S_2) = \min(\infty, 3) = 3\)
      • \(M(g_2, S_2) = \min(\infty, 3) = 3\)
    • Row 3 스캔: \(S_1\)과 \(S_3\)가 1을 가집니다. \(g_1(3)=0, g_2(3)=2\)를 기존 \(M\)값과 비교하여 갱신합니다.
      • \(S_1\) 업데이트: \(M(g_1, S_1) = \min(1, 0) = 0\), \(M(g_2, S_1) = \min(1, 2) = 1\)
      • \(S_3\) 업데이트: \(M(g_1, S_3) = \min(2, 0) = 0\), \(M(g_2, S_3) = \min(0, 2) = 0\)
  • [최종 서명 행렬 (Signature Matrix)]
    • 결과적으로 문서들의 최종 서명은 다음과 같이 도출됩니다 (강의 자료 원본의 OCR 깨짐을 교정하여 정확한 값으로 산출한 결과입니다).
Hash fn \(S_1\) \(S_2\) \(S_3\)
\(g_1\) 0 3 0
\(g_2\) 1 3 0

4. 문서를 벡터화하는 다른 기법들 (Other Ways to Vectorize)

  • 지금까지는 문서 내 토큰의 “존재 유무(1 or 0)”에만 기반하여 Jaccard 유사도를 구하는 방식(Minhashing)을 다루었습니다. 하지만 실제 텍스트 마이닝에서는 단어의 등장 빈도(Frequency)와 희귀성(Rarity)을 고려하는 방식이 널리 쓰이는데, 그 대표적인 방법이 TF-IDF(Term Frequency-Inverse Document Frequency)입니다.

4.1 TF-IDF의 수식적 정의

  • 전체 말뭉치(Corpus)에 존재하는 고유 단어의 개수를 \(M\)이라고 할 때, TF-IDF는 각 문서를 길이 \(M\)짜리의 실수형(Real-valued) 벡터 시그니처로 변환합니다. 문서 \(k\)의 \(i\)번째 원소는 \(TF_{ik} \times IDF_i\) 로 계산됩니다.
    • TF (Term Frequency): 단어가 해당 문서 내에서 얼마나 자주 등장했는지를 정규화한 값입니다. 특정 문서 내에서 가장 많이 등장한 단어의 빈도로 나누어 0~1 사이의 값으로 맞춥니다. \[TF_{ik} = \frac{f_{ik}}{\max_t f_{tk}}\] (여기서 \(f_{tk}\)는 문서 \(k\)에서 단어 \(t\)가 등장한 빈도입니다) .
    • IDF (Inverse Document Frequency): 단어가 전체 말뭉치에서 얼마나 희귀한지를 나타내는 패널티 지표입니다. \[IDF_i = \log \left( \frac{N}{n_i} \right)\]
      • 여기서 \(N\)은 전체 문서의 수, \(n_i\)는 단어 \(i\)가 포함된 문서의 수입니다.
      • 흔한 단어(예: “the”, “a”)는 \(n_i\)가 커져서 IDF 값이 0에 가까워집니다.

4.2 Minhash 서명과 TF-IDF 서명의 차이점 (Properties)

  • 두 기법은 본질적인 성격이 매우 다릅니다.
    • Minhashed Signatures: 정수형 벡터(Integer vectors)로 구성되며, 유사도는 두 벡터가 “정확히 동일한 정수 요소(same integer elements)를 얼마나 많이 공유하는가”로 측정됩니다.
    • TF-IDF Signatures: 실수형 벡터(Real-valued vectors)로 구성됩니다. 값이 연속적이므로 두 문서가 완벽히 일치하는 요소를 공유할 확률은 거의 없습니다. 따라서 코사인 거리(Cosine distance) 등 실수 기반의 거리 측정을 사용해야 하며, 이를 LSH에 적용하려면 Minhashing과는 완전히 다른 방식의 Locality-Sensitive Hash 함수가 필요해집니다.

5. 무엇이 “좋은 서명”인가? (Representation Learning)

  • 데이터 마이닝 분야를 넘어, 현대의 머신러닝(Machine Learning)은 문서를 단순한 규칙(Minhash, TF-IDF)이 아닌 데이터 학습을 통해 직접 시그니처를 만들어 내는 단계로 진화했습니다.

  • 이러한 분야를 표현 학습(Representation Learning)이라고 부르며, 여기서 생성된 고차원 실수형 시그니처 벡터를 우리는 보통 임베딩(Embeddings)이라고 부릅니다. ChatGPT와 같은 거대 언어 모델(LLMs)을 포함한 모든 현대적 NLP(자연어 처리) 시스템은 바로 이 임베딩 공간 위에서 작동합니다.
