1. Introduction

  • ліё нҸ¬мҠӨнҠёлҠ” лҢҖмҡ©лҹү лҚ°мқҙн„° мІҳлҰ¬ л°Ҹ лҚ°мқҙн„° л§ҲмқҙлӢқмқ„ мң„н•ң н•өмӢ¬ н”„л Ҳмһ„мӣҢнҒ¬мқё Apache Sparkмқҳ к°ңмҡ”(Overview)лҘј лӢӨлЈ№лӢҲлӢӨ. л¶„мӮ° мІҳлҰ¬ мӢңмҠӨн…ңмқҳ л°ңм „ кіјм •мқ„ мқҙн•ҙн•ҳкё° мң„н•ҙм„ңлҠ” кё°мЎҙмқҳ нҢЁлҹ¬лӢӨмһ„мқҙм—ҲлҚҳ н•ҳл‘Ў(Hadoop)кіј MapReduceмқҳ кө¬мЎ°лҘј лЁјм Җ м§ҡкі  л„ҳм–ҙк°Ҳ н•„мҡ”к°Җ мһҲмҠөлӢҲлӢӨ. кё°мЎҙ мӢңмҠӨн…ңмқҙ к°Җм§„ к·јліём Ғмқё л””мҠӨнҒ¬ I/O лі‘лӘ© нҳ„мғҒмқ„ нҢҢм•…н•ҳкі , мқҙлҘј л°©н–Ҙм„ұ л№„мҲңнҷҳ к·ёлһҳн”„(DAG, Directed Acyclic Graph)мҷҖ л©”лӘЁлҰ¬ кё°л°ҳ(In-memory) м—°мӮ°мқ„ нҶөн•ҙ м–ҙл–»кІҢ к·№ліөн–ҲлҠ”м§Җ к·ё л…јлҰ¬м Ғ нқҗлҰ„мқ„ л”°лқјк°Җ ліҙкІ мҠөлӢҲлӢӨ.

2. Hadoop FrameworkмҷҖ MapReduceмқҳ кө¬мЎ°

  • м•„нҢҢм№ҳ н•ҳл‘Ў(Hadoop)мқҖ лҢҖк·ңлӘЁ лҚ°мқҙн„°мқҳ л¶„мӮ° мІҳлҰ¬лҘј мң„н•ҙ м„Өкі„лҗң мҳӨн”ҲмҶҢмҠӨ н”„л Ҳмһ„мӣҢнҒ¬мһ…лӢҲлӢӨ. мҙҲм°Ҫкё° н•ҳл‘Ў мғқнғңкі„м—җм„ң лҚ°мқҙн„° мІҳлҰ¬ лЎңм§ҒмқҖ MapReduce м—”м§„кіј л§Өмҡ° к°•н•ҳкІҢ кІ°н•©(tightly coupled)лҗҳм–ҙ мһҲм—ҲмҠөлӢҲлӢӨ. н•ҳл‘Ў н”„л Ҳмһ„мӣҢнҒ¬лҠ” нҒ¬кІҢ м„ё к°Җм§Җ н•өмӢ¬ м»ҙнҸ¬л„ҢнҠёлЎң кө¬м„ұлҗ©лӢҲлӢӨ:
    • HDFS (Hadoop Distributed File System): кұ°лҢҖн•ң лҚ°мқҙн„°м…Ӣмқ„ м—¬лҹ¬ нҒҙлҹ¬мҠӨн„° л…ёл“ңм—җ л¶„мӮ°н•ҳм—¬ м ҖмһҘн•ҳл©°, лҚ°мқҙн„° мң мӢӨмқ„ л§үкё° мң„н•ҙ ліөм ңліё(Replication)мқ„ мң м§Җн•©лӢҲлӢӨ.
    • YARN (Yet Another Resource Negotiator): нҒҙлҹ¬мҠӨн„° лӮҙ м—¬лҹ¬ л…ёл“ңмқҳ лҰ¬мҶҢмҠӨлҘј кҙҖлҰ¬н•ҳкі  мһ‘м—… мҠӨмјҖмӨ„л§Ғмқ„ лӢҙлӢ№н•©лӢҲлӢӨ. нҳ„лҢҖ н•ҳл‘Ў мӢңмҠӨн…ңм—җм„ң вҖңл§ҲмҠӨн„° м»ЁнҠёлЎӨлҹ¬(Master controller)вҖқлқј н•ЁмқҖ мЈјлЎң мқҙ YARNмқ„ м§Җм№ӯн•©лӢҲлӢӨ.
    • Processing Engines: мӢӨм ң м—°мӮ° мһ‘м—…(Computation)мқ„ мҲҳн–үн•ҳлҠ” м—”м§„мңјлЎң, MapReduceмҷҖ Sparkк°Җ лҢҖн‘ңм Ғмһ…лӢҲлӢӨ.

3. MapReduceмқҳ к·јліём Ғ н•ңкі„ (Limitations of MapReduce)

  • MapReduceлҠ” лӢЁмҲңн•ҳкі  к°•л Ҙн•ҳм§Җл§Ң, ліөмһЎн•ң лҚ°мқҙн„° л§ҲмқҙлӢқмқҙлӮҳ лЁёмӢ лҹ¬лӢқ м•Ңкі лҰ¬мҰҳмқ„ мҲҳн–үн•ҳкё°м—җлҠ” лӘҮ к°Җм§Җ м№ҳлӘ…м Ғмқё кө¬мЎ°м Ғ н•ңкі„лҘј м§ҖлӢҲкі  мһҲмҠөлӢҲлӢӨ.

3.1 л””мҠӨнҒ¬ I/OлЎң мқён•ң мҳӨлІ„н—Өл“ң

  • MapReduceлҠ” л§ө(Map) нғңмҠӨнҒ¬мҷҖ лҰ¬л“ҖмҠӨ(Reduce) нғңмҠӨнҒ¬ мӮ¬мқҙмқҳ мӨ‘к°„ мғқм„ұл¬ј(Intermediate files)мқ„ л©”лӘЁлҰ¬к°Җ м•„лӢҢ л””мҠӨнҒ¬м—җ м ҖмһҘн•©лӢҲлӢӨ.

  • к°Ғ Map нғңмҠӨнҒ¬мқҳ м¶ңл Ҙ кІ°кіјлҠ” н•ҙлӢ№ мӣҢм»Ө(Worker) л…ёл“ңмқҳ лЎңм»¬ нҢҢмқј мӢңмҠӨн…ңм—җ кё°лЎқлҗ©лӢҲлӢӨ.

  • мһҘм җ(Pros): л©”мқё л©”лӘЁлҰ¬(RAM) мҡ”кө¬лҹүмқҙ м Ғм–ҙ, л©”лӘЁлҰ¬к°Җ л¶ҖмЎұн•ң нҷҳкІҪм—җм„ңлҸ„ м•Ҳм •м ҒмңјлЎң лҢҖмҡ©лҹү мһ‘м—…мқ„ мҲҳн–үн•  мҲҳ мһҲмҠөлӢҲлӢӨ.

  • лӢЁм җ(Cons): л§Ө Map нғңмҠӨнҒ¬к°Җ лҒқлӮ  л•Ңл§ҲлӢӨ л°ңмғқн•ҳлҠ” л””мҠӨнҒ¬ I/O мҳӨлІ„н—Өл“ңк°Җ л§үлҢҖн•©лӢҲлӢӨ. л§Ңм•Ҫ мӣҢм»Ө л…ёл“ңм—җ мһҘм• к°Җ л°ңмғқн•ҳл©ҙ н•ҙлӢ№ л””мҠӨнҒ¬м—җ м ҖмһҘлҗң мӨ‘к°„ нҢҢмқјм—җ м ‘к·јн•  мҲҳ м—ҶкІҢ лҗҳлҠ” л¬ём ңлҸ„ л°ңмғқн•©лӢҲлӢӨ.

3.2 л°ҳліө(Iterative) м•Ңкі лҰ¬мҰҳ м Ғмҡ©мқҳ м–ҙл ӨмӣҖ

  • нҠ№нһҲ лЁёмӢ лҹ¬лӢқ мөңм Ғнҷ”(мҳҲ: Gradient Descent) м•Ңкі лҰ¬мҰҳмқҖ лҸҷмқјн•ң лҚ°мқҙн„°м—җ лҢҖн•ҙ м—°мӮ°мқ„ мҲҳмӢӯ~мҲҳл°ұ лІҲ л°ҳліөн•ҙм•ј н•©лӢҲлӢӨ.
    • MapReduceлҠ” ліөмһЎн•ң мһ‘м—…мқ„ мҲҳн–үн•  л•Ң, лҚ°мқҙн„° ліөм ң(Replication)мҷҖ л””мҠӨнҒ¬ мһ…м¶ңл ҘмңјлЎң мқён•ҙ л§үлҢҖн•ң мҳӨлІ„н—Өл“ңлҘј л°ңмғқмӢңнӮөлӢҲлӢӨ.
    • кІ°кіјм ҒмңјлЎң л§ҺмқҖ мөңмӢ  м•Ңкі лҰ¬мҰҳл“Өмқ„ лӢЁмҲңнһҲ Mapкіј ReduceлқјлҠ” л‘җ к°Җм§Җ лӢЁкі„л§ҢмңјлЎң н‘ңнҳ„н•ҳкі  м„ӨлӘ…н•ҳкё°лҠ” л§Өмҡ° м–ҙл өмҠөлӢҲлӢӨ.

4. м°Ём„ёлҢҖ лҢҖм•Ҳ: Workflow Systemкіј DAG

  • MapReduceмқҳ кІҪм§Ғлҗң мІҙмқё кө¬мЎ°(Map \(\rightarrow\) Reduce \(\rightarrow\) Map \(\rightarrow \dots\))лҘј к·№ліөн•ҳкё° мң„н•ҙ м ңм•Ҳлҗң кІғмқҙ л°©н–Ҙм„ұ л№„мҲңнҷҳ к·ёлһҳн”„(DAG) кё°л°ҳмқҳ Workflow Systemмһ…лӢҲлӢӨ.

4.1 DAG(Directed Acyclic Graph)лЎңмқҳ нҷ•мһҘ

  • мқҙ мӢңмҠӨн…ңм—җм„ңлҠ” м—°мӮ°л“Өмқҙ DAG нҳ•нғңлЎң кө¬м„ұлҗ©лӢҲлӢӨ. DAG н–үл ¬ \(G = (V, E)\)м—җм„ң м •м җ \(V\)лҠ” к°Ғ нғңмҠӨнҒ¬лҘј, к°„м„  \(E\)лҠ” лҚ°мқҙн„°мқҳ нқҗлҰ„мқ„ мқҳлҜён•©лӢҲлӢӨ.
    • лӢЁмқј мһ…л Ҙл§Ң л°ӣлҚҳ н•ңкі„лҘј лІ—м–ҙлӮҳ, н•ҳлӮҳ мқҙмғҒмқҳ лӢӨмӨ‘ мһ…л Ҙмқ„ л°ӣлҠ” н•ЁмҲҳл“Өмқ„ н—Ҳмҡ©н•©лӢҲлӢӨ.
    • л§ҲмҠӨн„° м»ЁнҠёлЎӨлҹ¬лҠ” м „мІҙ DAGлҘј л¶„м„қн•ҳм—¬ м—¬лҹ¬ нғңмҠӨнҒ¬ к°„мқҳ мһ‘м—…мқ„ нҡЁмңЁм ҒмңјлЎң л¶„н• н•ҳкі  н• лӢ№н•©лӢҲлӢӨ.

4.2 лӮҙкІ°н•Ём„ұмқ„ мң„н•ң м°ЁлӢЁ нҠ№м„ұ (Blocking Property)

  • л¶„мӮ° мӢңмҠӨн…ңм—җм„ң л…ёл“ң мӢӨнҢЁлҠ” н•„м—°м Ғмһ…лӢҲлӢӨ. Workflow мӢңмҠӨн…ңмқҖ Blocking PropertyлҘј нҶөн•ҙ кІ°н•Ё ліөмӣҗл Ҙ(Fault-resilient model)мқ„ нҷ•ліҙн•©лӢҲлӢӨ.
  • нҠ№м • м—°мӮ°мқҖ м„ н–үлҗҳлҠ” лӘЁл“  мһ‘м—…(Predecessors)мқҙ мҷ„лЈҢлҗҳм–ҙм•јл§Ң мӢӨн–үмқ„ мӢңмһ‘н•  мҲҳ мһҲмҠөлӢҲлӢӨ.
  • л”°лқјм„ң л§Ңм•Ҫ м–ҙл–Ө нғңмҠӨнҒ¬к°Җ мӢӨнҢЁн–ҲлӢӨл©ҙ, к·ё нӣ„н–ү мһ‘м—…(Successors)мқҖ м•„м§Ғ мӢңмһ‘мЎ°м°Ё н•ҳм§Җ м•Ҡм•ҳмқҢмқ„ л…јлҰ¬м ҒмңјлЎң ліҙмһҘн•  мҲҳ мһҲмҠөлӢҲлӢӨ.
  • ліөкө¬ л©”м»ӨлӢҲмҰҳ: л§ҲмҠӨн„° м»ЁнҠёлЎӨлҹ¬лҠ” мӢӨнҢЁн•ң нғңмҠӨнҒ¬лҘј лӢӨлҘё кұҙк°•н•ң л…ёл“ңм—җм„ң мһ¬мӢңмһ‘мӢңнӮөлӢҲлӢӨ. мҳҲлҘј л“Өм–ҙ, мң„ DAG лҸ„мӢқм—җм„ң л…ёл“ң \(g\)к°Җ мӢӨнҢЁн•ңлӢӨл©ҙ, м„ н–ү л…ёл“ңмқё \(f\)лҘј лӢӨмӢң мӢӨн–үн•ң л’Ө к·ёлһҳн”„мқҳ нқҗлҰ„мқ„ л”°лқј \(g\)лҘј мһ¬мӢӨн–үн•ҳл©ҙ мҷ„лІҪн•ң ліөкө¬к°Җ к°ҖлҠҘн•©лӢҲлӢӨ.

5. Apache Spark: Overview & Ecosystem

  • Apache SparkлҠ” лӢЁмҲңнһҲ MapReduceлҘј лҢҖмІҙн•ҳлҠ” кІғмқ„ л„ҳм–ҙ, нӣЁм”¬ лҚ” мң м—°н•ҳкІҢ м„Өкі„лҗң к°ҖмһҘ мқёкё° мһҲлҠ” Workflow Systemмһ…лӢҲлӢӨ. UC BerkeleyмҷҖ Databricksм—җм„ң к°ңл°ңлҗҳм–ҙ нҳ„мһ¬лҠ” Apache мҶҢн”„нҠёмӣЁм–ҙ мһ¬лӢЁм—җм„ң мң м§ҖліҙмҲҳн•ҳкі  мһҲмҠөлӢҲлӢӨ.

5.1 Sparkмқҳ мЈјмҡ” нҳҒмӢ  нҸ¬мқёнҠё

  • кё°мЎҙ MapReduceм—җ лҢҖл№„н•ҳм—¬ Sparkк°Җ м¶”к°Җ/к°ңм„ н•ң н•өмӢ¬ мҡ”мҶҢлҠ” лӢӨмқҢкіј к°ҷмҠөлӢҲлӢӨ:
    • мҙҲкі мҶҚ лҚ°мқҙн„° кіөмң (Fast data sharing): мӨ‘к°„ м—°мӮ° кІ°кіјлҘј л””мҠӨнҒ¬м—җ м ҖмһҘн•ҳлҠ” кІғмқ„ н”јн•ҳкі  л©”лӘЁлҰ¬м—җ мң м§Җн•©лӢҲлӢӨ.
    • лҚ°мқҙн„° мәҗмӢұ(Data Caching): лЁёмӢ лҹ¬лӢқ(ML) м•Ңкі лҰ¬мҰҳмІҳлҹј лҸҷмқјн•ң мҝјлҰ¬к°Җ л°ҳліөлҗҳлҠ” кІҪмҡ°, лҚ°мқҙн„°лҘј л©”лӘЁлҰ¬м—җ мәҗмӢң(Cache)н•ҳм—¬ м„ұлҠҘмқ„ к·№лҢҖнҷ”н•©лӢҲлӢӨ.
    • мқјл°ҳнҷ”лҗң мӢӨн–ү к·ёлһҳн”„ (General DAGs): Mapкіј ReduceліҙлӢӨ нӣЁм”¬ н’Қл¶Җн•ҳкі  лӢӨм–‘н•ң н•ЁмҲҳ м—°мӮ°мқ„ м§Җмӣҗн•ҳлҠ” DAG мӢӨн–ү м—”м§„мқ„ м ңкіөн•©лӢҲлӢӨ.
    • Hadoop нҳёнҷҳм„ұ: н•ҳл‘Ў HDFS л“ұ кё°мЎҙ л¶„мӮ° нҢҢмқј мӢңмҠӨн…ңкіј мҷ„лІҪнһҲ нҳёнҷҳлҗ©лӢҲлӢӨ.

6. RDD (Resilient Distributed Dataset)

  • SparkлҘј мқҙн•ҙн•ҳлҠ” лҚ° мһҲм–ҙ к°ҖмһҘ мӨ‘мҡ”н•ң н•өмӢ¬ м•„мқҙл””м–ҙмқҙмһҗ мӨ‘мӢ¬ лҚ°мқҙн„° м¶”мғҒнҷ” к°ңл…җмқҙ л°”лЎң RDDмһ…лӢҲлӢӨ.
    • м •мқҳ: лӢЁмқј нғҖмһ…мқҳ л ҲмҪ”л“ңл“ӨлЎң мқҙлЈЁм–ҙм§„, нҢҢнӢ°м…”лӢқ(л¶„н• )лҗң лҚ°мқҙн„° м»¬л үм…ҳмһ…лӢҲлӢӨ.
    • мқјл°ҳнҷ”: MapReduceм—җм„ң мӮ¬мҡ©лҗҳлҚҳ лӢЁмҲңн•ң Key-Value мҢҚмқҳ к°ңл…җмқ„ нӣЁм”¬ лҚ” лІ”мҡ©м ҒмңјлЎң мқјл°ҳнҷ”н•ң нҳ•нғңмһ…лӢҲлӢӨ.
    • нҠ№м„ұ:
      • лҚ°мқҙн„° мІӯнҒ¬ лӢЁмң„лЎң м „мІҙ нҒҙлҹ¬мҠӨн„°м—җ л¶„мӮ°лҗҳм–ҙ м ҖмһҘлҗҳл©°, мғқм„ұлҗң нӣ„м—җлҠ” ліҖкІҪн•  мҲҳ м—ҶлҠ” мқҪкё° м „мҡ©(Read-only/Immutable) мҶҚм„ұмқ„ к°Җм§‘лӢҲлӢӨ.
      • мһ‘м—… мҶҚлҸ„ н–ҘмғҒмқ„ мң„н•ҙ л©”мқё л©”лӘЁлҰ¬(RAM)м—җ мәҗмӢң(Cached)лҗ  мҲҳ мһҲмҠөлӢҲлӢӨ.
    • мғқм„ұ л°©лІ•: л¶„мӮ° нҢҢмқј мӢңмҠӨн…ң(DFS)м—җм„ң м§Ғм ‘ лҚ°мқҙн„°лҘј мқҪм–ҙмҳӨкұ°лӮҳ, лӢӨлҘё RDDм—җ ліҖнҷҳ(Transformation)мқ„ к°Җн•ЁмңјлЎңмҚЁ мғҲлЎӯкІҢ мғқм„ұлҗ©лӢҲлӢӨ.
    • лҸҷмқјн•ң м—°мӮ° лЎңм§Ғмқ„ м»¬л үм…ҳ лӮҙмқҳ лӘЁл“  мҡ”мҶҢм—җ лі‘л ¬лЎң м Ғмҡ©н•  л•Ң мөңкі мқҳ м„ұлҠҘмқ„ л°ңнңҳн•©лӢҲлӢӨ.

7. Sparkмқҳ н”„лЎңк·ёлһҳл°Қ лӘЁлҚё: Transformation vs. Action

  • Spark н”„лЎңк·ёлһЁмқҖ RDDмқҳ м—°мӮ° нқҗлҰ„мқ„ м„Өкі„н•ҳлҠ” кіјм •мқҙл©°, нҒ¬кІҢ л‘җ к°Җм§Җ мң нҳ•мқҳ м—°мӮ°мңјлЎң лӮҳлү©лӢҲлӢӨ.

7.1 Transformations (ліҖнҷҳ)

  • лӢӨлҘё RDDлЎңл¶Җн„° мғҲлЎңмҡҙ RDDлҘј кө¬м¶•н•ҳлҠ” м—°мӮ°мһ…лӢҲлӢӨ.
    • нҠ№м§•: лі‘л ¬ мІҳлҰ¬к°Җ к°ҖлҠҘн•ң кІ°м •лЎ м Ғ(Deterministic) м—°мӮ°мһ…лӢҲлӢӨ.
    • мў…лҘҳ: map, filter, join, union, distinct л“ұмқҙ мЎҙмһ¬н•©лӢҲлӢӨ.
    • м§Җм—° нҸүк°Җ (Lazy Evaluation) \(\star\):
      • к°ҖмһҘ мӨ‘мҡ”н•ң нҠ№м§•мңјлЎң, Transformation мҪ”л“ңлҘј мһ‘м„ұн•ҳлҚ”лқјлҸ„ мҰүмӢң мӢӨн–үлҗҳм§Җ м•Ҡмңјл©°, л’Өмқҙм–ҙ м„ӨлӘ…н•  вҖңActionвҖқмқҙ нҳём¶ңлҗҳкё° м „к№Ңм§ҖлҠ” м–ҙл– н•ң мӢӨм ң кі„мӮ°(Computation)лҸ„ мҲҳн–үлҗҳм§Җ м•ҠмҠөлӢҲлӢӨ.
      • мҲҳмӢқмңјлЎң н‘ңнҳ„н•ҳл©ҙ мҙҲкё° мғҒнғң \(RDD_0\)м—җм„ң \(T(RDD_i) \rightarrow RDD_{i+1}\)мқҳ м—°мӮ° кі„нҡҚ(Lineage)л§Ң к·ёлһҳн”„ нҳ•нғңлЎң лҲ„м Ғлҗ©лӢҲлӢӨ.

7.2 Actions (н–үлҸҷ)

  • мӢӨм ңлЎң мҠӨмјҖмӨ„лҹ¬лҘј кө¬лҸҷн•ҳм—¬ кі„мӮ°мқ„ к°•м ң(force calculations)н•ҳкі  кІ°кіјк°’мқ„ л°ҳнҷҳн•ҳкұ°лӮҳ мҷёл¶Җ м ҖмһҘмҶҢлЎң лҚ°мқҙн„°лҘј лӮҙліҙлӮҙлҠ” м—°мӮ°мһ…лӢҲлӢӨ.
    • нҠ№м§•: Actionмқҙ нҳём¶ңлҗҳлҠ” мҲңк°„, мқҙм „м—җ лҲ„м Ғлҗң м§Җм—°лҗң Transformationл“Өмқҙ н•ң лІҲм—җ мөңм Ғнҷ”лҗҳм–ҙ лі‘л ¬лЎң мӢӨн–үлҗ©лӢҲлӢӨ.
    • мў…лҘҳ: count, collect, reduce, save л“ұмқҙ мһҲмҠөлӢҲлӢӨ.
